# GRPD 接触模块 V5.0 终极演进版蓝图：四正交轴与摩擦本构体系

在经过深度的理论探讨后，我们发现现有的接触本构严重受制于“几何起伏（齿轮效应）”和“动/静力学求解器下摩擦力响应缺失”的物理瓶颈。老旧的高度耦合结构（如现存的 KinematicContact）已无法满足复杂多变的物理需求。

为了将 GRPD 打造成能比肩顶级显/隐式商用软件的粒子法求解器，我们的接触架构必须经历一次脱胎换骨的升维重组，这正式确立了 **V5.0 终极接触机制的“四大独立正交轴（A+B+C+D）隔离架构”**。

---

## 1. 终极四正交轴隔离架构 (Quad-Axis Decoupling)

未来的接触循环将如汽车装配线一般被切分为四大高度解耦、可横向乐高拼装的算子多态组件：

### 📍 轴 A：搜索与空间预处理 (Search Domain Engine)
- **核心理念**：雷达索敌。永远遵循 **"Slave 找 Master"** 的单向搜索原则。
- **职责**：只负责构建 Spatial Hash Grid 或 KD-Tree，划定 Pinball 圈半径，挑出发生潜在碰撞的邻居粒子。决不插手几何投影与力学计算。

### 🔪 轴 B：接触几何与运动学评判 (Contact Kinematics & Normal Evaluator)
- **核心理念**：提取接触面的几何形态（法向、切向）与运动学变量（相对位移增量、等效速度等），消除离散粒子的几何空间起伏。
- **职责**：将轴 A 发来的粒子群，通过多态策略算出绝对平滑稳定的法向 $\vec{n}$、穿透量 $d$，为力学计算提供唯一数据抓手 `ContactContext`。
  1. **NTN (Node-To-Node 连线法)**：最粗泛原始的几何连线投影，适合不需要考究连续平滑静摩擦的高能极速破碎。
  2. **NTS Kernel (核函数虚拟面法)**：采用密度梯度缝合出完美零噪音的连续平滑曲面，用于材料极度扭曲大变形，能完全托住最严苛的隐式历史弹簧摩擦算法。
  3. **NTS Precomputed (预计算平面贴砖法)**：刚体撞击与固定墙壁的零开销神技！调用预先缓存的纯正几何法线配合局部平面投影，以 O(1) 的极速算力消除穿透量颠簸，且原汁原味地保留尖锐刀口的几何锋利度。
  4. **NTC (Node-To-Cloud 加权云聚合)**：即原 Kinematic 算法中的几何骨架。通过网格体积加权，算出综合群体的等效法向和穿透，将一对多关系降维为一对一虚拟包裹。

### ⚙️ 轴 C：法向排斥本构定律 (Normal Force Law)
- **核心理念**：单纯处理“如何抗拒物体间的法向穿透挤压”，杜绝物质渗透。
- **职责**：它是个只认数据的计算插件，接纳轴 B 送来的穿透量 $d$，不关心是谁送来的、也不关心摩擦方向。
  1. **StandardPenalty**：纯位移相关的经典刚度罚函数法。
  2. **Silling**：带有短程力体积空间积分特性的近场动力学特色罚函数。
  3. **Viscous**：附带相对逼近速度阻尼的粘性防震垫法则。

### 🛡️ 轴 D：切向阻滞本构定律 (Friction Law Modeler)
- **核心理念**：专攻切平面上的滑动阻力与静止咬合，且其物理极限永远被轴 C 提供的法向排斥力 $F_N$ 死死钳制。
- **职责**：作为和轴 C 平行的独立装配插件，它彻底解决了接触力学中难度最大、策略最丰富的切向摩擦机制：
  1. **ExplicitImpulse (冲量截断动摩擦)**：沿袭现有代码神技，基于瞬时相对极速的截断急刹，不需要记忆数组包袱。完美搞定极速大冲大撞下的瞬间制止（子弹穿甲等显式动力学主打场合）。
  2. **TanhRegularized (速度正则化双曲正切平滑摩擦)**：无数据包袱神级偏方，曲线无限丝滑无极降噪，配合 NTS 大面使用，属于金属冲压、塑性切削等大滑移连续接触算例的首选。
  3. **ImplicitStickSlip (拉格朗日隐式历史弹簧摩擦)**：静力学终极归宿模型。必须挂载外部哈希表来记录上一步的残余应力与微弱错位，利用试探力与屈服面返回映射。无论 ADR 再怎么抽清速度，它的真实位移感永远能稳稳拽住物体，实现真正物理零滑移僵持。

## 2. 积木化配置文件格式 (YAML Assembly)
这一套体系打通后，你的 `.yaml` 会呈现出搭积木般的工业美感，你可以任意拼装上万种可能性：

```yaml
ContactPairs:
  - Name: "BulletToWall"
    SearchMethod: "HashGrid"            # 轴 A: 索敌机制 (找谁碰了)
    NormalMethod: "KinematicCloud"      # 轴 B: 几何特征与等效面抓取法 (NTC加权聚合, 或NTS面投影)
    ForceLaw: "StandardPenalty"         # 轴 C: 法向排斥力法则 (对抗几何穿透的刚度本构)
    FrictionLaw: "TanhRegularized"      # 轴 D: 切向阻滞法则 (防切向侧滑的专用引擎)
    FrictionCoeff: 0.3                  # 被 C 和 D 共同钳制的极限天花板
    V_Ref: 0.001                        # D 轴 Tanh 摩擦定律所独有的定制参数
```

## 3. 面向 V5.0 的开发阶段 (Roadmap)

基于这套完整的理论支撑，我们可以制定接下来的手术排期：

- **第一阶段 (肢解老旧战甲，基建打平)**：将旧有臃肿的 `KinematicContact` 彻底重构拆解剖腹。它的自带索网格能力归回轴 A；它的宏观加权平均法提纯入轴 B 重生为 `NTCEvaluator`；它的高速脉冲截断动摩擦被剥离入轴 D 成为首批 `ExplicitImpulseLaw`。此阶段不引入任何新物理机制，仅向四正交轴架构完美过渡兼容。
- **第二阶段 (引爆轴 D 插件库：拥抱 Tanh 与 StickSlip)**：火力全开扩编独立的摩擦力宇宙。快速实装无历史包袱的极顺滑 `TanhRegularizedLaw` (大形变滑行通解，搭配 NTS 极爽)；攻克技术巅峰点，挂载外部哈希图字典实现 `ImplicitStickSlipLaw` (解决被封死在 ADR 动能清零诅咒下的死磕型静摩擦计算)。
- **第三阶段 (NTS 神级法向算子下放)**：向轴 B 库里面灌入核函数密度梯度投影体系。NTS 将为娇贵的轴 D（尤其是弹簧历史截断）提供无与伦比的光滑绝对切平面支撑，根除所有搓衣板噪波，迈入殿堂级接触模拟。
