# Smoke-Tester 排查与修复摘要 (Walkthrough)

## 变更总结
- **问题**：在严格通过 MCP 调度进行冒烟测试时，`ansys-server` 的所有工具调用瞬间报 `EOF` 崩溃，导致无法执行基准提取。
- **根因定位**：排查发现 `ansys-mcp-server/src/generator.py` 和 `service.py` 内部包含普通的 `print()` 语句。由于 MCP 微服务严格依赖 `stdout` 传输 JSON-RPC 协议，这些纯文本打印污染了 JSON 数据流，导致大模型宿主的解析器崩溃并强行断开服务连接。
- **修复措施**：为避免破坏原有的业务逻辑，且遵从用户不引入顶层 Python 封装的架构意愿，已通过正则替换将所有散落的 `print(...)` 重定向至错误日志通道（`file=sys.stderr`）。
- **影响范围**：仅修复了日志输出的流向，原有的解耦式、三段式验证链路被完全保留。
