# 数值验证 Playbook

## 目标

每个 CAE 改动都应至少有一个最小验证闭环：能运行、能解释、能失败。

## 推荐验证

- 材料：单点或小粒子单轴拉伸，检查应力应变曲线、屈服、卸载和状态变量。
- Field pipeline：检查字段是否存在、维度正确、可输出。
- Kernel：检查力方向、能量趋势、邻域数量和稳定性。
- TimeIntegrator：检查 CFL clamp、步长截断、载荷步时间。
- Contact：检查 gap、normal、接触力方向和 active 粒子。
- Fracture：检查阈值触发、断键、损伤输出。
- IO：检查 VTK 是否含目标字段，ParaView 中命名是否清楚。

## 输出格式

验证说明应包含：

- 测试目标。
- 输入文件或示例路径。
- 运行命令。
- 期望结果。
- 失败判据。
- 是否需要人工看 VTK/日志。
